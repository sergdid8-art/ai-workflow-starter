# Бизнес-контекст: Coffee Place

> ⚠️ **Вымышленный пример.** Coffee Place, все имена, контакты, адреса, ИНН, цифры, Telegram-хендлы — придуманы для демонстрации. Любое совпадение с реальным бизнесом / человеком случайно.
>
> Сценарий: небольшая сеть из 3 кофеен в региональном центре + мобильное приложение для заказа.

## Карта папок

- [`company/`](./company/) — история, команда, ценности, юр. структура
- [`products/`](./products/) — меню, тарифы подписки, программа лояльности
- [`audience/`](./audience/) — 3 аватара (утренние рабочие, студенты, семьи с детьми)
- [`goals/`](./goals/) — открытие 4-й точки к концу года + запуск подписки
- [`economics/`](./economics/) — unit-экономика кофе, средний чек, LTV, структура расходов
- [`marketing/`](./marketing/) — каналы, воронка через приложение, конкуренты
- [`assets/`](./assets/) — брендинг (тёмно-зелёный + охра), отзывы

## Правило работы

Клод читает **только нужный файл** при задаче, не загружает весь `.business/` сразу. Экономия токенов.

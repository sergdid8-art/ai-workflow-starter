# План: бэкенд подписки Coffee Monthly

**Дата:** 2026-04-18
**Автор:** Марат + Клод
**Статус:** в работе

## Цель

Добавить в бэкенд мобильного приложения модуль «Подписка» — хранение активных подписок, списание балансов напитков при заказе, автоматическое продление через Альфа-Банк рекуррент.

## Контекст

Подписка Coffee Monthly — ключевая фича на Q2 2026. Нужен работающий бэкенд к 15 мая (фронт делается параллельно).

Что уже есть: приложение на React Native + Node.js бэкенд + PostgreSQL. Оплата разовых заказов через Альфа-Банк эквайринг.

Чего нет: модели данных для подписки, рекуррент-списания, списания балансов при заказе.

## Фазы

### Фаза 1. Модель данных (~1 день)
- [x] Таблица `subscriptions` (user_id, tier, drinks_total, drinks_used, started_at, expires_at)
- [x] Таблица `subscription_transactions` (лог списаний и пополнений)
- [x] Миграция Prisma

### Фаза 2. API подписки (~2 дня)
- [x] POST `/api/subscription/create` — создание подписки + инициация оплаты
- [x] POST `/api/subscription/webhook/alfa` — вебхук от Альфы об успешной оплате
- [ ] GET `/api/subscription/status` — текущая подписка юзера
- [ ] POST `/api/subscription/cancel` — отмена автопродления

### Фаза 3. Интеграция со списанием (~1.5 дня)
- [ ] При заказе через приложение: если у юзера активная подписка + есть баланс — списывать из подписки, не брать деньги
- [ ] Если подписка истекла — показать напоминание об обновлении
- [ ] Лимит напитков не может уйти в минус

### Фаза 4. Рекуррент (~1 день)
- [ ] Настроить рекуррент через Альфа-Банк API
- [ ] Cron-задача раз в день: проверить подписки, которым пора продлевать, инициировать платёж
- [ ] Email/push об успешном/неуспешном продлении

### Фаза 5. Тесты (~0.5 дня)
- [ ] Unit-тесты логики списания
- [ ] E2E сценарий через Playwright: купить подписку → сделать заказ → проверить списание баланса

## Критерии готовности

- [ ] Юзер может оформить Starter/Regular/Barista подписку в приложении
- [ ] При заказе напитка баланс уменьшается на 1
- [ ] При истечении подписки — автосписание через Альфу или email «не прошла оплата, обнови карту»
- [ ] Тесты зелёные
- [ ] На staging прошёл полный пользовательский сценарий

## Риски

- **Альфа-Банк рекуррент работает с задержкой 1-2 дня** — подписка не должна «гаснуть» в эти дни. Буфер 3 дня на нулевом балансе.
- **Юзер оплатил но вебхук не пришёл** — нужен фолбэк: crontask проверяет pending платежи через API Альфы.
- **Юзер отменил карту** — корректная обработка: уведомление + выключение подписки через 7 дней.

## Открытые вопросы

- Делать ли refund если юзер отменил подписку после 1 заказа? → Решение Марата: нет, подписка неотменяемая в первые 7 дней, потом только отмена продления.
- Лимит одновременных подписок на юзера? → Решение: 1 подписка, новая заменяет старую.

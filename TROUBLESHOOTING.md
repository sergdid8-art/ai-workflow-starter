# Troubleshooting

Решения типовых проблем при настройке и работе с этим шаблоном.

## Git и SSH

### Git не установлен

Скачай и установи Git for Windows:
https://git-scm.com/download/win

Проверь:
```powershell
git --version
```

### SSH не работает

Проверь, что ключ добавлен в GitHub и агент SSH доступен:
```powershell
ssh -T git@github.com
```

Если ошибка — повторно сгенерируй ключ и добавь публичную часть в GitHub.

### `git commit` говорит “please tell me who you are”

Настрой локально для проекта:
```powershell
git config user.name "Твоё Имя"
git config user.email "email@example.com"
```

## VS Code

### Папка `.business/` не видна в дереве

VS Code скрывает скрытые папки по умолчанию. Проверь настройки в `.vscode/settings.json` и перезагрузите окно.

### AI-ассистент не читает правила проекта

Обычно это значит, что:
- агент открыл не тот контекст;
- в чате слишком длинный контекст;
- файл `QWEN.md` или `CLAUDE.md` не был прочитан.

В таком случае начни запрос с фразы:
> сначала прочитай `QWEN.md`, `CLAUDE.md` и `.business/INDEX.md`

## Проект и контекст

### Контекст потерян

Если задача уже обсуждалась, но агент снова “с нуля”:
- открой `plans/` и `retrospectives/`;
- проверь `.business/INDEX.md`;
- не начинай новый чат без необходимости.

### Идея слишком расплывчатая

Сначала уточни:
- цель;
- критерий успеха;
- ограничения;
- кто будет пользоваться результатом.

## Безопасность

### Случайно закоммитил секреты

Сразу:
1. удалите секрет из истории и из рабочей ветки;
2. ротируйте ключи;
3. проверь `git status` и `.gitignore`.

## Деплой и запуск

Для деплоя, публикации и интеграции смотри папку `prompts/launch/`.

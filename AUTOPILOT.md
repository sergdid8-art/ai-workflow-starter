# Project onboarding

> Этот файл — вспомогательный сценарий для быстрого старта работы с проектом. Его цель — перевести идею в понятный, проверяемый процесс.

## 1. Понять задачу

Проверь, что уже известно:
- что именно нужно сделать;
- зачем это нужно;
- кто получит результат;
- какой критерий успеха.

Если данных мало — сначала задай 1–3 точных вопроса.

## 2. Зафиксировать контекст

Заполни или проверь файлы:
- `.business/INDEX.md`
- `.business/company/`
- `.business/products/`
- `.business/audience/`
- `.business/goals/`

## 3. Сформировать минимальный план

Для нетривиальной задачи создай план в `plans/`.

В плане укажи:
- цель;
- шаги;
- ограничения;
- риски;
- что будет считаться успешным.

## 4. Выбрать путь реализации

Сначала выбери решение с минимальной сложностью, которое решает задачу. Не делай крупного рефакторинга без причины.

## 5. Реализовать по шагам

- один шаг — один результат;
- после каждого шага — проверка;
- если обнаружена проблема — зафиксируй её и скорректируй план.

## 6. Проверить результат

Проверь:
- код/текст/структуру;
- влияние на существующий проект;
- крайние случаи;
- отсутствие скрытых побочных эффектов.

## 7. Зафиксировать вывод

После завершения задачи:
- обнови план;
- добавь запись в `retrospectives/`;
- кратко зафиксируй, что было сделано, что сработало и что стоит улучшить.

## 8. Коммит и передача дальше

Перед коммитом проверь:
- нет ли секретов и локальных данных;
- в `git status` только нужные изменения;
- проект в состоянии, пригодном для продолжения.

Этот процесс подходит для Qwen, Copilot и других AI-ассистентов, если нужен предсказуемый и экономичный рабочий цикл.

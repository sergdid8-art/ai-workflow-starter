# plans/ - технические планы реализации

## Правила

1. **Один план = одна функция.** Не «сделать проект», а «добавить форму оплаты».
2. **Имя файла:** `YYYY-MM-DD-название-функции.md`
3. **План делится на фазы.** У каждой фазы статус `[ ]` или `[x]`.
4. **В конце плана** - итоговый блок: реализован целиком или нет, что осталось.
5. **Любой агент обязан актуализировать план** после каждой сессии: отмечать сделанное, дописывать что изменилось.

## Зачем это нужно

Новый чат Клода не помнит, что было сделано в предыдущем. **План - помнит.** Когда открываешь новый чат для продолжения задачи, Клод читает план, видит где остановились, что сделано, что нет - и продолжает оттуда.

Без плана большие задачи разваливаются на 3-й день работы.

## Как работать с планом

1. **Создай** план в начале задачи: `plans/2026-04-23-payment-form.md` по шаблону `TEMPLATE.md`.
2. **Критикуй** через субагентов: возьми промпт `prompts/methodology/plan-critique.md`.
3. **Реализуй** по фазам. После каждой фазы - ставь `[x]`.
4. **В конце** закрой блок «Итог» - что получилось, что не сделано, что осталось на потом.

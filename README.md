# AI Workflow Starter

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](./LICENSE)

Повторяемая операционная система для работы над продуктами с AI-ассистентами. Шаблон помогает превратить хаотичный старт в предсказуемый процесс: от понимания задачи до проверки результата.

## Для чего это нужно

Большинство проблем в проектах появляются не из-за отсутствия кода, а из-за:
- размытых требований;
- потери контекста между сессиями;
- бессмысленных больших правок;
- слабой проверки результата;
- отсутствия единого способа работать с AI.

AI Workflow Starter решает это через простую и понятную структуру:
1. сначала контекст;
2. потом план;
3. затем минимальная реализация;
4. после — проверка и ретроспектива.

## Что вы получаете

- `.business/` — единый контекст проекта: зачем, для кого и зачем это делается;
- `QWEN.md` и `CLAUDE.md` — правила поведения агента и рабочий режим;
- `plans/` и `retrospectives/` — дисциплина для планирования и обучения на опыте;
- `prompts/` — готовые сценарии для запуска, анализа, ревью и деплоя;
- базовые шаблоны, безопасность и структура папок — чтобы не терять время на рутину.

## Кому это подходит

- founders и solo-builders;
- продуктовым командам;
- агентам и AI-ассистентам в повседневной работе;
- командам, которые хотят быстро масштабировать рабочие процессы без хаоса.

## Быстрый старт

1. Склонируй репозиторий или используй его как основу для нового проекта.
2. Заполни `.business/` — это твой “второй мозг” проекта.
3. Открой проект в VS Code.
4. Запусти AI-ассистента и скажи: “сначала прочитай `QWEN.md` и `.business/INDEX.md`, затем предложи план”.
5. Работай по циклу: план → реализация → проверка → ретроспектива.

## Структура репозитория

```text
project-root/
├── QWEN.md
├── CLAUDE.md
├── AUTOPILOT.md
├── TROUBLESHOOTING.md
├── .business/
├── plans/
├── retrospectives/
├── prompts/
├── templates/
└── .vscode/
```

## Принципы работы

- Не делай больших изменений без контекста.
- Не начинай реализацию без ясной цели и критерия успеха.
- Если данных мало — сначала задай 1–3 точных вопроса.
- Проверяй результат фактами, а не ожиданиями.
- После завершения задачи фиксируй вывод в ретроспективе.

## Безопасность

- Не коммить секреты и ключи.
- Не храните чувствительные данные в коде.
- Используй `.env` и локальные конфиги только для локального окружения.
- Перед крупной задачей — проверь план и риски.

## Как это выглядит на практике

Этот шаблон помогает перейти от идеи к рабочему процессу без потери контекста:
- понять задачу;
- зафиксировать требования;
- разбить её на шаги;
- выполнять по одному результату за раз;
- проверять и документировать вывод.

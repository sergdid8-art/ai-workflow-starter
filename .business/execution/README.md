# execution/ - операционное планирование

Папка для еженедельной работы: план на неделю и бэклог.

## Что тут лежит

- `backlog.md` - задачи без даты, куда класть идеи, которые всплыли не вовремя
- `YYYY-WNN.md` - файлы недели (создаются по мере надобности, по номеру ISO-недели)
- `TEMPLATE-week.md` - шаблон файла недели

Фокус месяца живёт в `../goals/monthly.md` (не здесь) - чтобы стратегия и операционка не расползались.

## Триггерные фразы

Сказал Клоду - Клод сделал:

| Фраза | Действие |
|---|---|
| «давай спланируем неделю» | Прочитает `../goals/monthly.md`, создаст `YYYY-WNN.md` с задачами по дням |
| «добавь задачу X» | На конкретный день - в файл недели. Без даты - в `backlog.md` |
| «что на этой неделе?» | Покажет файл текущей недели |
| «итоги недели» | Заполнит секцию «Итог недели» |
| «перенеси X на следующую» | Уберёт из текущей, добавит в следующую |

Полный промпт планирования недели - `prompts/methodology/weekly-planning.md`.

## Правила

- Не больше 5 ключевых результатов на неделю
- Задачи конкретные («написать 3 поста», не «поработать над контентом»)
- Еженедельная петля: план → выполнение → итог → коррекция месячных целей

# Импорт существующего проекта в `.business/`

**Когда брать:** у тебя **уже есть** рабочий проект (сайт / SaaS / приложение) и ты хочешь навести в нём порядок по методологии второго мозга.
**Что делает:** Клод читает твой текущий код, структуру папок, README, документацию и сам заполняет `.business/` на основе того что нашёл.
**Результат:** `.business/` под твой существующий проект без ручного интервью.

## Когда использовать vs интервью

- **Используй этот промпт**, если у тебя уже есть код, и Клод может по нему понять что за продукт, кто аудитория, какая модель монетизации.
- **Используй интервью** (`prompts/setup/04-business-interview.md`), если проект на стадии идеи или у тебя есть только общее представление.

## Промпт

```prompt
У меня уже есть рабочий проект. Код лежит в корне этого репо (или по пути: [ВСТАВЬ ПУТЬ если другой]).

Задача: на основе существующего кода, документации, README, package.json, структуры папок - заполни папку .business/ по методологии второго мозга.

ДЕЙСТВУЙ ПО ШАГАМ:

1. Изучи проект. Прочитай:
   - README.md (если есть) - обычно там основная информация
   - package.json - видно стек, зависимости, название
   - Структура папок - видно архитектуру
   - Код главных страниц (index, лендинг, главная) - видно offer и аудиторию
   - Модели данных (Prisma schema, DB migrations) - видно бизнес-сущности
   - Файлы документации (docs/, .md файлы) - если есть

2. Составь предварительную картину проекта: что делает, для кого, какая модель монетизации (по коду видно есть ли платежи, подписки, тарифы).

3. Задай мне 5-7 вопросов, чтобы закрыть пробелы. ПО ОДНОМУ за раз. К каждому предлагай 3-4 варианта ответа на основе того что ты увидел в коде.

4. После ответов заполни файлы в .business/:
   - company/about.md, company/values.md, company/legal.md
   - products/overview.md, products/pricing.md (если видел цены в коде)
   - audience/avatar.md, audience/segments.md (по тому какие сегменты видны в коде)
   - goals/ - спроси у меня
   - economics/ - если видел данные о ценах/подписках - заполни, остальное спроси
   - marketing/ - channels.md, competitors.md, content.md - спроси
   - assets/brand-guidelines.md - если есть файлы стилей (CSS, tailwind config) - вытащи цвета/шрифты оттуда

5. В конце обнови INDEX.md с оглавлением всей папки .business/.

6. Сделай Reality Check: найди 2-3 пробела в моём проекте (что в коде выглядит незавершённым, противоречивым, или без явной бизнес-причины).

Отвечай на русском. По одному вопросу, не допрос.
```

## После импорта

- Проверь файлы `.business/` - что-то Клод угадал, что-то нет. Поправь.
- Добавь то, что в коде не отражено (цели на год, приватные конкуренты, прогноз).
- Убедись что `.business/` в `.gitignore` - теперь это приватные данные.

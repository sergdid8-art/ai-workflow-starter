# Критика плана через 3 параллельных субагента

**Когда брать:** когда Клод написал первый вариант плана в `plans/YYYY-MM-DD-название.md` и ты готов проверить его прежде чем начинать реализацию.
**Что делает:** запускает 3 субагента параллельно - каждый смотрит на план со своей стороны (техника, бизнес, UX). Возвращает общий отчёт с правками.
**Результат:** улучшенный план с закрытыми дырами, которые ты бы не увидел один.

## Почему это работает

Один агент в одном чате не может «посмотреть на свою работу со стороны». Новый чат + параллельные субагенты - это и есть «взгляд со стороны», который ловит 80% проблем **до того** как ты начал кодить.

## Рабочий цикл

1. Клод пишет первичный план в `plans/YYYY-MM-DD-название.md`
2. Ты **копируешь ссылку на файл**, открываешь **новый чат**
3. В новом чате запускаешь промпт ниже
4. Возвращаешься к исходному плану и обновляешь с учётом критики
5. **Только после этого** - реализация

## Промпт

```prompt
У меня есть первичный план реализации: [путь к файлу, например plans/2026-04-23-payment-form.md].

Задача:
1. Прочитай план целиком.
2. Прочитай .business/INDEX.md и связанные файлы - чтобы понять контекст проекта.
3. Запусти 3 параллельных субагента на глубокое исследование:
   - агент 1: проверь техническую реализуемость каждой фазы, найди риски и подводные камни
   - агент 2: проверь соответствие плана бизнес-целям из .business/, найди пробелы и противоречия
   - агент 3: сделай ревью UX/пользовательского сценария - где пользователь может застрять, что непонятно
4. Собери их выводы в один отчёт с конкретными правками: что добавить, что убрать, что переформулировать.
5. Покажи мне отчёт. Я решу какие правки принять, и попрошу тебя обновить файл плана.

Всегда отвечай на русском.
```

## Что делать с отчётом

- **Критика обоснована** → обнови план
- **Критика промах** (агент не понял контекст) → отметь себе, пропусти
- **Критика поднимает новый вопрос** → не фикси сейчас, добавь в «Открытые вопросы» плана

## Правило

Не делай все правки разом. Пройдись по отчёту пункт за пунктом. На каждую правку принимай осознанное решение «да/нет/позже».

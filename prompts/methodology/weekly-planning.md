# Планирование недели

**Когда брать:** в начале недели (понедельник утро) или в конце предыдущей недели (воскресенье вечер).
**Что делает:** Клод читает твои месячные цели, предлагает задачи на неделю, создаёт файл недели в `.business/execution/`.
**Результат:** `.business/execution/YYYY-WNN.md` с задачами по дням и 3-5 ключевыми результатами.

## Принцип

Цели на год → квартал → месяц → **неделя** → день. Неделя = последнее звено между стратегией и реальными действиями. Без недельного плана месячные цели - просто желания.

## Промпт: спланировать неделю

```prompt
Давай спланируем эту неделю.

1. Прочитай .business/goals/monthly.md - фокус месяца.
2. Прочитай .business/goals/quarterly.md - цели квартала.
3. Прочитай .business/execution/backlog.md если есть - что там накопилось.
4. Предложи мне 3-5 ключевых результатов на эту неделю, которые двигают проект к месячным целям.
5. По каждому результату предложи задачи с распределением по дням недели (пн-пт, вых опционально).
6. Создай файл .business/execution/YYYY-WNN.md (используй текущую ISO-неделю) по этому формату:

```
# Неделя YYYY-WNN (DD.MM - DD.MM)

## Фокус недели
[одно предложение: что главное]

## Ключевые результаты
1. [результат 1]
2. [результат 2]
3. [результат 3]

## План по дням

### Понедельник
- [ ] задача

### Вторник
- [ ] задача

...

## Итог недели
[заполняется в воскресенье]
- Что сделано:
- Что не сделано и почему:
- Что перенесено:
- Уроки:
```

7. Покажи финальный файл. Спроси, что поправить.
```

## Промпт: добавить задачу

```prompt
Добавь задачу: [описание задачи].
```

Клод сам определит куда положить:
- **На конкретный день** - в файл текущей недели в нужный день
- **Без даты / на будущее** - в `.business/execution/backlog.md`

## Промпт: итоги недели

В воскресенье вечером:

```prompt
Закроем неделю. Заполни секцию «Итог недели» в .business/execution/YYYY-WNN.md:
1. Что сделано (отметь фактически закрытые задачи).
2. Что не сделано и почему.
3. Что перенести на следующую неделю.
4. Уроки - что я понял за эту неделю.

Затем предложи перенос несделанных задач: какие в следующую неделю, какие в бэклог.
```

## Правила

- **Не больше 5 ключевых результатов.** Шесть = ноль.
- **Задачи = конкретные действия.** Не «поработать над контентом», а «написать 3 поста для Telegram-канала».
- **Петля:** месячные цели → неделя → дни → итоги недели → коррекция месячных целей.

## Триггеры (Клод знает их из CLAUDE.md)

| Ты говоришь | Клод делает |
|---|---|
| «давай спланируем неделю» | промпт выше |
| «добавь задачу» | в текущую неделю или бэклог |
| «что на этой неделе?» | покажет файл недели |
| «итоги недели» | заполнит «Итог недели» |
| «перенеси X на следующую» | уберёт из текущей, добавит в следующую |

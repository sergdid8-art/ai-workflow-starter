# Генерация CLAUDE.md под твой проект

**Когда брать:** после заполнения `.business/` через интервью.
**Что делает:** создаёт `CLAUDE.md` в корне проекта - главный файл с правилами работы, который Клод читает первым перед каждым действием.
**Результат:** `CLAUDE.md` под твой стек и тип проекта, не больше 200 строк.

## Зачем

`.business/` - это **знания** (зачем делаем проект). `CLAUDE.md` - это **правила** (как Клод работает со знаниями). Без `CLAUDE.md` Клод каждый раз «придумывает» подход заново. С ним - следует единому методу.

⚠️ **Не больше 200 строк** - иначе сжигаешь токены каждый запрос.

## Промпт

```prompt
Создай в корне проекта файл CLAUDE.md. Основа - шаблон в templates/CLAUDE.md.tmpl.

Действуй так:

1. Прочитай templates/CLAUDE.md.tmpl - это заготовка с плейсхолдерами.
2. Прочитай .business/INDEX.md и все файлы внутри .business/ - чтобы понять мой проект.
3. Заполни все плейсхолдеры в шаблоне ({{PROJECT_NAME}}, {{PROJECT_ONE_LINER}}, {{STACK}}, {{LANGUAGE_RULE}}) на основе моих ответов из .business/.
4. Если каких-то данных не хватает - задай мне уточняющие вопросы ПО ОДНОМУ с 3-4 вариантами ответа.
5. Запиши итоговый файл в корень проекта как CLAUDE.md.
6. Проверь что файл не больше 200 строк. Если больше - сократи, убирая воду. НЕ удаляй разделы про план для каждой функции и про рефлексию.
7. Покажи мне итоговый файл и спроси что доработать.

Важно: два правила должны остаться в CLAUDE.md обязательно:
- «План для каждой новой функции в plans/»
- «Ретроспектива в конце каждой сессии в retrospectives/»

Всегда отвечай на русском.
```

## Проверка

После создания задай Клоду микрозадачу: «создай файл HELLO.md с приветствием». Клод должен:
1. Сначала прочитать `CLAUDE.md` и `.business/INDEX.md`
2. Создать план в `plans/YYYY-MM-DD-hello.md`
3. Выполнить задачу
4. Обновить план - отметить фазу как `[x]`
5. Написать ретро в `retrospectives/`

Если какой-то шаг пропущен - значит `CLAUDE.md` написан плохо. Попроси Клода доработать и повтори тест.

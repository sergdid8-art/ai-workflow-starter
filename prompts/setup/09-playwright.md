# Playwright MCP - Клод управляет браузером

**Когда брать:** когда у тебя уже есть работающий сайт (локально или в сети) и ты хочешь, чтобы Клод мог его тестировать сам.
**Что делает:** подключает Playwright MCP - Клод открывает настоящий браузер, тыкает на кнопки, заполняет формы, делает скриншоты, проверяет мобильную вёрстку.
**Результат:** `.mcp.json` или настройки MCP в проекте + работающая автоматизация браузера.

## Зачем

- Сделал лендинг - Клод сам открывает его, проверяет формы, шлёт скриншоты с ошибками.
- Надо протестировать чужой сайт - Клод прогоняет сценарий и пишет отчёт.
- Регрессия после правки - автоматический прогон, чтобы убедиться что ничего не сломал.

⚠️ **Совет:** у некоторых Playwright работает нестабильно. Если у тебя полезут странности - не зависай, поставь альтернативу (Chrome DevTools MCP) и иди дальше.

## Установка

```prompt
Установи Playwright MCP в мой текущий проект Claude Code.

1. Найди официальный пакет (Microsoft Playwright MCP или @modelcontextprotocol/playwright).
2. Проведи аудит безопасности - это браузер с полным доступом, важно понять что он делает.
3. Если безопасно - установи ЛОКАЛЬНО для этого проекта (в .mcp.json или настройки Claude Code этого проекта, НЕ глобально).
4. После установки покажи как его вызывать, какие команды доступны (navigate, click, screenshot, fill и т.д.).
5. Сделай проверочный прогон: открой google.com, сделай скриншот, покажи мне.
```

## Твой первый автотест

Когда у тебя появится сайт:

```prompt
Используй Playwright MCP. Открой мой локальный сайт по адресу http://localhost:3000.

1. Сделай скриншот первого экрана.
2. Прокрути страницу вниз до конца - скриншот каждого блока.
3. Найди главную форму (кнопка «Купить» / «Оставить заявку»), заполни тестовыми данными (имя: Тест, email: test@example.com), нажми кнопку.
4. Что произошло после отправки? Скриншот результата.
5. Переключи viewport на мобилку (iPhone 14). Повтори прогон.
6. Собери отчёт: что работает, где баги, какие блоки выглядят плохо на мобилке.
```

Клод пойдёт и всё сделает. Получишь тестовый отчёт со скриншотами - и ни строчки кода сам не написал.

## Важно про токены

Каждый подключённый MCP добавляет ~2000 токенов в контекст **каждого** сообщения. Не подключай десятки MCP «на всякий случай» - держи только те, которыми активно пользуешься.

# Создать папку `plans/` с правилами

**Когда брать:** если папка `plans/` отсутствует в проекте (например, ты склонировал не шаблон, а чужой репо).
**Что делает:** создаёт папку `plans/` с README и TEMPLATE.
**Результат:** `plans/README.md` + `plans/TEMPLATE.md`.

## Зачем

Папка `plans/` - память между сессиями. Новый чат Клода не помнит прошлое. План - помнит.

**Один план = одна функция.** Не «весь проект», а «добавить форму оплаты».

## Промпт

```prompt
Создай в корне моего проекта папку plans/. Внутри создай два файла:

### plans/README.md

Коротко опиши (10-15 строк):
1. Что такое папка plans/ - это место для технических планов реализации функций.
2. Правило: один план = одна функция.
3. Формат имени файла: YYYY-MM-DD-название-функции.md
4. Каждый план делится на фазы. У каждой фазы статус [ ] или [x].
5. В конце плана - итоговый блок: реализован целиком или нет, что осталось.
6. Любой агент обязан актуализировать план после каждой сессии.

### plans/TEMPLATE.md

Шаблон файла плана с разделами:
- Цель (одно предложение)
- Контекст (откуда задача, что есть, чего нет)
- Фазы (с чекбоксами)
- Критерии готовности
- Риски
- Открытые вопросы
- Итог (заполняется в конце)

После создания покажи мне дерево корня проекта (CLAUDE.md, .business/, plans/, retrospectives/), чтобы я убедился что всё на месте.
```

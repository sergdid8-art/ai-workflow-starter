# 4 запрета безопасности

**Когда брать:** сразу после `02-hooks.md`, обязательно перед включением Bypass Permissions.
**Что делает:** блокирует 4 опасных типа команд на уровне `.claude/settings.json`. Даже если ты случайно скажешь «удали всё» - Клод остановится и переспросит.
**Результат:** `.claude/settings.json` с секцией `permissions.deny`.

## Что именно запрещаем

| Команда | Почему опасно |
|---|---|
| `rm -rf` и массовое удаление | Можно случайно стереть диск |
| `curl` + `export` переменных | Утечка секретов наружу |
| Чтение `.env` целиком | API-ключи и пароли попадают в транскрипт |
| `wget` с подставленными переменными | Экспорт данных наружу |

## Промпт

```prompt
Настрой мне в .claude/settings.json правила безопасности. ЗАПРЕТИ:

1. Команды вида `rm -rf` и любое массовое удаление файлов или папок без явного подтверждения от меня.
2. Связку `export` + `curl` - любую отправку переменных окружения наружу через curl.
3. Чтение файлов .env и любых файлов с секретами, ключами, паролями (читать .env можно только через grep по конкретной переменной).
4. `wget` с подставленными переменными - любую отправку моих данных наружу через wget.

Используй секцию permissions.deny (или эквивалентную секцию в актуальной версии Claude Code). За образцом посмотри templates/claude-settings.json.example - там есть пример секции permissions.deny с 9 правилами (4 базовых + расширения).

Перед изменением - покажи текущий файл. После изменения - покажи итоговый .claude/settings.json и объясни каждое правило одной строкой.
```

## Проверка

Попроси Клода выполнить `rm -rf tmp/` в тестовой папке - он должен остановиться и попросить явное подтверждение.

Попроси прочитать `.env` - он должен отказаться или использовать `grep`.

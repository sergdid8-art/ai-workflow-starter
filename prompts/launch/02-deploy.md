# Задеплоить проект в интернет

**Когда брать:** код на GitHub, хочешь чтобы сайт был доступен по ссылке.
**Что делает:** поднимает сайт на одной из платформ, настраивает автодеплой при каждом пуше.
**Результат:** публичная ссылка на твой сайт.

## Выбор платформы

| Платформа | Кому подходит | Особенности |
|---|---|---|
| **Vercel** | Next.js / React / Vue / статика | Бесплатно для простых проектов, быстрее всех |
| **Railway** | Международные проекты с БД | $5/мес, сайт + БД в одном месте |
| **Amvera** | Проекты под российскую аудиторию | Серверы в РФ, закон 152-ФЗ, автопроксирование к Claude API |
| **Свой сервер** | Максимум контроля | Нужна админская работа, не для новичков |

**Не знаешь что выбрать:**
- Российская аудитория → Amvera (серверы в РФ)
- Международная → Vercel для простых сайтов, Railway если нужна БД
- Пилотный проект без БД → Vercel (бесплатно)

## Путь A. Vercel

```prompt
Задеплой мой проект на Vercel.

1. Зарегистрируйся через GitHub (если нужно - объясни что нажать на vercel.com).
2. Импортируй мой GitHub-репозиторий.
3. Настрой автодеплой - чтобы при каждом пуше в main сайт обновлялся.
4. Если нужны переменные окружения - спроси меня какие значения вставить, но НЕ коммить их в репо.
5. Покажи финальную ссылку на работающий сайт.
```

## Путь Б. Railway (международный, с БД)

```prompt
Задеплой мой проект на Railway.

1. Зарегистрируйся на railway.app (если нужно - объясни что нажать).
2. Подключи мой GitHub-репозиторий.
3. Настрой автодеплой - чтобы при каждом пуше сайт обновлялся.
4. Если нужна конфигурация (Procfile, Dockerfile, переменные окружения) - создай.
5. Покажи финальную ссылку на работающий сайт.
```

## Путь В. Amvera Cloud (Россия)

Amvera - российский аналог Railway. Серверы в Москве. Закон 152-ФЗ (ПДн россиян на серверах РФ) покрыт. Автопроксирование к Claude и OpenAI API - трафик идёт через зарубежный IP, VPN на сервере не нужен.

При регистрации дают 111 руб. бесплатно - хватит попробовать.

```prompt
Задеплой мой проект на Amvera Cloud.

1. Объясни пошагово как зарегистрироваться на amvera.ru и создать проект.
2. Добавь Amvera как remote в git.
3. Создай amvera.yml (если нужен).
4. Запуши проект на Amvera.
5. Покажи финальную ссылку на работающий сайт.
```

## Проверка

Открой ссылку на телефоне. Если сайт загружается - ты в интернете. Поздравляю.

## Если упал деплой

```prompt
Деплой упал с ошибкой: [вставь текст ошибки или скриншот].

Разберись почему и исправь. В 90% случаев это отсутствие переменных окружения, отсутствие `build`-команды в package.json или проблема с версией Node.
```

В 90% случаев Клод сам починит.

## Правило

**Не застревай на одном этапе дольше 20 минут.** Не получается деплой? Попробуй другую платформу. Главное - сайт в интернете.

# Подключить приём платежей

**Когда брать:** когда есть рабочий сайт и готов принимать деньги.
**Что делает:** интегрирует платёжную систему - ты выбираешь какую, Клод делает код.
**Результат:** на сайте работает кнопка оплаты, деньги приходят тебе.

## Выбор платёжки

| Платёжка | Кому подходит | Особенности |
|---|---|---|
| **CloudPayments** | Россия, SaaS, подписки | Красивый виджет, Alpha Pay, Сбер Pay, СБП |
| **Робокасса** | Россия, разовые продажи | Проще всего, всё подключено сразу |
| **Тинькофф** | Россия, экосистема Т-Банка | Только карты + Тинькофф Pay |
| **ЮKassa** | Россия, популярный выбор | Широкая сеть способов оплаты |
| **Stripe** | Международный | Стандарт рынка, подписки, payment links |

**Не знаешь что выбрать:**
- Россия + разовые продажи → Робокасса
- Россия + подписки/SaaS → CloudPayments
- Международка → Stripe

## Методология (одна для любой платёжки)

1. Клод находит SDK и документацию
2. Клод изучает интеграцию
3. Клод пишет код
4. Клод пишет тебе инструкцию что настроить в ЛК платёжки
5. Клод проверяет безопасность (ключи только в `.env`)

## Промпт

```prompt
Я хочу подключить [CloudPayments / Робокассу / Stripe / ЮKassa] к моему проекту.

1. Найди официальный SDK и документацию API.
2. Изучи документацию, разберись в интеграции.
3. Добавь кнопку оплаты на сайт. Контекст: [опиши за что платят, например: «Оплатить отчёт - 490 руб.» или «Купить подписку - 990 руб./мес»].
4. Напиши мне ПОШАГОВУЮ инструкцию - что сделать в личном кабинете платёжки: зарегистрироваться, создать проект, получить ключи.
5. Все секретные ключи сохрани в .env файл. Ничего секретного в коде.
6. Настрой webhook для уведомления об успешной оплате.

Пока не подключай реальные ключи - сначала разберёмся с настройкой.
```

## Результат

У тебя в проекте готовый код для приёма оплаты + инструкция что делать дальше. Когда зарегистрируешься в платёжке и получишь ключи - вставишь в `.env`, задеплоишь, всё заработает.

## Безопасность

⚠️ Критично:
- **Никогда** не коммить ключи платёжки в git. Только в `.env`.
- **Никогда** не клади ключи в клиентский JS (они попадают в браузер). Только на серверной стороне.
- **Перед push на прод** - ротейт тестовые ключи на продовые.
- **Верифицируй webhook** через подпись от платёжки (иначе кто угодно может подделать «оплата прошла»).

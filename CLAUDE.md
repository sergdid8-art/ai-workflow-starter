# Project operating rules

> Этот файл читается первым любым AI-ассистентом, который работает с этим репозиторием.

## Главная цель

Сделать работу над проектом предсказуемой, дешёвой и безопасной. Не тратить лишние токены на бессмысленные действия, не терять контекст и не скрывать неопределённости.

## Принципы работы

1. Сначала прочитай `.business/INDEX.md` и `QWEN.md`.
2. Если задача нетривиальная, сначала сформулируй цель, ограничения, критерий успеха и риски.
3. Не делай большие изменения без плана.
4. Работай маленькими шагами: один шаг — один результат.
5. Перед тем как говорить “готово”, проверь результат фактами.
6. Сохраняй память проекта в `plans/` и `retrospectives/`.
7. Если данных недостаточно, задай 1–3 точных вопроса, а не выдумывай.
8. Не коммить секреты, ключи, персональные данные и локальные конфиги.

## Рекомендуемый цикл

- Планирование: `plans/`
- Реализация: по шагам, без лишних правок
- Проверка: тесты, сборка, ручная проверка, diff
- Ретроспектива: `retrospectives/`

## Что делать при новой задаче

1. Пойми, что требуется.
2. Сформулируй минимальный рабочий план.
3. Определи риски и ограничение по времени.
4. Реализуй самое небольшое решение, которое закрывает задачу.
5. Проверь вывод и зафиксируй результат.

## Контроль стоимости и риска

- Не генерируй лишний код без цели.
- Не делай несколько альтернатив сразу.
- Не запускай крупный рефакторинг без явной причины.
- Если изменение влияет на продукт, пользователя или инфраструктуру — сначала проверь контекст.

## Формат ответа

Для сложных задач отвечай в таком формате:
1. Короткий вывод
2. Что будет сделано
3. Основные риски
4. План по шагам
5. Как проверить результат

## Язык общения

По умолчанию используй русский. Если пользователь просит другой язык — переходи на него.

# retrospectives/ - рефлексии после каждой сессии

## Правила

1. **После каждой рабочей сессии** — пиши ретроспективу. Даже если сессия была короткой.
2. **Имя файла:** `YYYY-MM-DD_HH-MM_краткое-название.md`
3. **Формат** — см. [TEMPLATE.md](./TEMPLATE.md). 5 пунктов, без воды.

## Зачем это нужно

Ретроспектива — это передача опыта между сессиями. Новый чат агента не помнит прошлое. Ты сам через неделю тоже не помнишь. Ретроспектива — короткий файл, который можно прочитать за 30 секунд и быстро восстановить контекст.

## Формат (5 пунктов)

1. **Задача** — что было поставлено
2. **Как решал** — подход, инструменты, промпты, сколько времени
3. **Решил?** — да / нет / частично
4. **Что можно было лучше** — если бы делал заново, что изменил
5. **Как было / как стало** — до и после, одной строкой

## Пример

См. [TEMPLATE.md](./TEMPLATE.md) как заготовку. Первое реальное ретро появится после тестовой задачи и станет живым примером для будущих проектов.
